Capital Two Team

10/10/2020

Development

Structure of our repository

For the initial setup, each team member should have access to all repositories in the CapitalTwo organization. Each member should be an admin so as to have access to all repositories and freedom to create more if needed.

Components

Weekly-update – This repository contains weekly goals set by the team in the first meeting of the week. It will also contain weekly updates or notes to help with the documentation of the progress. A new folder will be created for each week. Team member responsible for maintaining this: Ritika

Main-Site – This repository will contain all tools, function page, homepage for the main website for the web application. The site is hosted at capitaltwo.ga or www.capitaltwo.ga. Team member responsible for maintaining this: Peter

Website-Content – This is a private repository containing the files that will be published on the knowledge passage. Team member responsible for maintaining this: Ritika

Recommend structure: (topic is the name of topic)

  • Topic
    • stage 1
      • topic-stage1.md
      • topic-stage1.json
    • stage 2
      • topic-stage2.md

Knowledge-Passage – The place to host knowledge.capitaltwo.ga, our knowledge passages. Generated automatically, please do NOT modify

Capstone-Wiki – The place for our wiki page, www.capitaltwo.ga . It contains our development process and weekly update. Generated automatically, please do NOT modify

  • Update the Weekly-update repository

  • Decide the main image for homepage
  • Optional: Create a function page, and get start button in the home page would direct to this page.
